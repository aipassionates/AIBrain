# mcp-servers/

Custom MCP servers written for this project. Each is a Node.js stdio server wrapped by supergateway for HTTP.

**Canonical source is here.** Live running copies are currently at `C:\tools\` until `infra/scripts/migrate-mcp-servers.ps1` is run.

---

## Servers

### media-mcp

yt-dlp wrapper exposing 5 tools:
- `video_info` — metadata without downloading
- `extract_transcript` — subtitles/captions → .vtt + plain text
- `download_video` — H.264/AAC MP4 at max 480p (AV1 explicitly avoided — unplayable on Windows)
- `download_audio` — MP3 extraction
- `list_downloads` — browse D:\media\ directory

Config: all paths and quality settings via env vars in PM2 config (no code changes needed).
Public: `https://media.passionate.agency/mcp`

### google-analytics-mcp

Google Analytics 4 + Search Console via OAuth2. Tools:
- `ga4_run_report` — flexible GA4 report (any dimensions + metrics)
- `gsc_query_performance` — Search Console performance data
- `gsc_list_sites` — list accessible GSC properties

Auth: `credentials.json` (from Google Cloud Console) + `token.json` (generated by `node auth.js`).
Both are gitignored. On a new machine: get credentials.json from Google Cloud Console → run `node auth.js` → token auto-refreshes thereafter.
Public: `https://analytics.passionate.agency/mcp`

## Migration (run once)

```powershell
.\infra\scripts\migrate-mcp-servers.ps1
```

Copies source from C:\tools\, installs deps at new location, updates PM2 to use new paths.

## Adding a new custom MCP server

1. Create `mcp-servers/<name>/index.js` using `@modelcontextprotocol/sdk`
2. Add `package.json` with dependencies
3. Create `infra/pm2/<name>.config.cjs` using the supergateway pattern
4. Follow `skills/add-new-mcp.md` for tunnel + Cowork wiring
